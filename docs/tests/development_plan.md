# Pathway 全链路 ETL 流水线开发计划

本文档基于 `docs/PathWay/plan.md` 制定，旨在详细规划 Pathway ETL 流水线的开发流程与里程碑。

## 阶段 1：核心算子工厂 (Operator Factory)
**目标**: 构建基于 Pathway `pw.Table` API 的动态调用基础，实现算子函数的字符串映射。

- **任务 1.1**: 设计算子注册机制
  - 实现装饰器或注册类，用于注册 Pathway 算子。
  - 建立字符串名称到算子函数的映射表。
- **任务 1.2**: 封装基础算子
  - 封装常用的 `pw.io` 输入输出算子。
  - 封装基础的数据转换算子（filter, map, reduce 等）。
- **任务 1.3**: 实现动态调用工厂
  - 开发 `OperatorFactory` 类，支持通过名称获取算子实例。
  - 实现参数透传机制，支持动态配置算子参数。

## 阶段 2：数据源接入与注册中心 (Data Source Integration)
**目标**: 打通现有数据源服务与 Pathway 执行引擎，实现多源数据接入。

- **任务 2.1**: 注册中心桥接器
  - 开发适配器，通过 `Source_ID` 从 Python 注册服务获取连接凭证（FTP/DB/API）。
  - 实现凭证的安全传递与解析。
- **任务 2.2**: 实时连接器封装
  - **FTP/文件系统**: 封装 `pw.io.fs`，支持通过配置接入远程/本地文件。
  - **数据库 CDC**: 封装 `pw.io.gorm` (或类似 CDC 连接器)，实现数据库变更捕获。
  - **Web/API**: 封装 `pw.io.http`，支持 Webhook 和 API 轮询。
- **任务 2.3**: 结构化预处理器
  - 开发 CSV 解析器：自动识别分隔符与表头。
  - 开发 JSON 解析器：支持嵌套结构的展平或提取。
  - 开发 Markdown 解析器：提取元数据与正文内容。

## 阶段 3：DAG 解析与构建 (DAG Parser)
**目标**: 实现从前端配置到可执行 DAG 的转换引擎。

- **任务 3.1**: 节点工厂 (Node Factory)
  - 实现 `DataSourceNode` 工厂：处理输入源配置。
  - 实现 `TransformNode` 工厂：处理中间转换逻辑。
  - 实现 `SinkNode` 工厂：处理输出目标配置。
- **任务 3.2**: DAG 解析逻辑
  - 定义 DAG 的 JSON 配置规范。
  - 实现拓扑排序与依赖分析，构建执行顺序。
  - 处理 Input/Output 的连接约束。
- **任务 3.3**: 参数校验与类型系统
  - 实现参数校验器，确保配置符合算子要求。
  - 实现类型检查，确保节点间数据流的 Schema 兼容性（如：DB Table 存在性校验）。

## 阶段 4：动态执行环境 (Dynamic Runner)
**目标**: 动态构建并运行 Pathway 计算图。

- **任务 4.1**: 执行器 (Runner) 实现
  - 开发 `PipelineRunner`，接收解析后的 DAG。
  - 动态构建 `pw.Table` 调用链。
  - 封装 `pw.run()` 执行入口，支持不同运行模式（阻塞/非阻塞）。
- **任务 4.2**: 运行时监控
  - 集成 Pathway 的监控指标。
  - 实现简单的日志捕获与状态上报。

## 阶段 5：前端可视化编排 (Visual Orchestrator)
**目标**: 优化前端拖拽界面，实现所见即所得的 ETL 流程配置。

- **任务 5.1**: 自定义节点库开发
  - **DataSourceNode**: 集成数据源选择下拉框。
  - **TransformNode**: 动态渲染算子参数配置表单。
  - **SinkNode**: 配置输出目标与格式。
- **任务 5.2**: 连线逻辑与交互
  - 实现端口（Handle）的类型限制（Source 仅输出，Sink 仅输入）。
  - 实现连线校验，防止循环依赖或类型不匹配。
- **任务 5.3**: 属性面板 (Property Panel)
  - 实现动态属性加载（例如：选择 DB 后异步加载 Table 列表）。
  - 支持复杂参数配置（JSON 编辑器、代码片段等）。
- **任务 5.4**: 实时状态反馈
  - 集成 WebSocket 客户端。
  - 在节点上通过不同颜色或图标显示运行状态（Pending, Running, Success, Error）。

## 阶段 6：多模态数据处理与图谱存储 (Multimodal & Graph)
**目标**: 结合 LightRAG 方案，实现多模态数据的图谱构建与实时处理。

- **任务 6.1**: 多模态图谱存储
  - 利用 `pw.Table` API 对接图数据库。
  - 实现多模态数据（文本、图像、视频）的节点与边构建逻辑。
  - 结合 LightRAG 实现基于图的检索增强生成。
- **任务 6.2**: 实时流处理与落库
  - 接入实时流数据源（社交媒体、传感器）。
  - 实现多模态数据的实时清洗与特征提取。
  - 建立实时数据到图数据库的写入通道。
