基于pathway驱动的全链路 ETL 流水线开发计划
阶段1：基于 Pathway 的 pw.Table API，编写一个动态调用工厂，支持根据字符串名映射到对应的算子函数。
阶段2：数据源接入与注册对接
利用您已有的数据源服务，打通“凭证”到“流”的通道。
注册中心桥接器： * 编写代码让 Pathway 执行引擎能通过 Source_ID 从您的 Python 注册服务中调取 FTP/DB/API 凭证。
实时连接器封装： * 封装 pw.io.fs (FTP 映射)、pw.io.gorm (数据库 CDC)、pw.io.http (爬虫 Webhook)。
结构化预处理器： * 开发针对不同文件格式（CSV, JSON, Markdown）的自动解析逻辑。
阶段3：实现一个基于 DAG 的解析器，能够将用户在前端拖拽的节点配置解析为可执行的 DAG 结构。
节点工厂： * 为每个节点类型（如：TransformNode, SinkNode）实现一个工厂类，负责根据配置创建对应的算子函数。
DAG 解析逻辑： * 解析用户配置，构建 DAG 结构，同时处理节点之间的 Input/Output 约束。
参数校验与类型转换： * 对节点配置中的参数进行校验，确保类型匹配（如：数据库 Table 必须是已注册的）。
阶段4：动态执行环境
动态执行环境： * 实现一个 Runner，能根据解析出的 DAG 动态构建 pw.Table 的调用链并执行 pw.run()。
阶段5：前端拖拽画布开发
基于已经构建的ETL流水线页面优化用户操作界面。
自定义节点库： * 开发 DataSourceNode (含下拉选择列表)、TransformNode (含参数配置项)、SinkNode。
连线逻辑与校验： * 实现 Handle (Input/Output) 的连接约束（例如：Source 节点不能有输入点）。
配置面板 (Property Panel)： * 实现点击节点时，动态从后端加载参数（如：选定数据库后自动加载 Table 列表）。
状态实时反馈： * 集成 WebSocket，在节点上显示处理进度或错误状态。
阶段6：多模态数据处理与落库
1. 基于现有项目的lightRAG方案实现图谱存储与查询
   - 利用pw.Table API，将解析后的多模态数据（如：文本、图像、视频）存储到图数据库中。
   - 实现基于图的查询，支持多模态数据的检索与分析。
2. 实现多模态数据的实时处理与落库
   - 利用pw.Table API，将实时流数据（如：社交媒体、传感器数据）处理为多模态数据。
   - 实现将处理后的多模态数据实时落库到图数据库中。
