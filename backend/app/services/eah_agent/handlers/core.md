handlers/ 目录是 eah_agent 服务层的业务执行枢纽。它承接来自 core/control_plane.py 分发的请求，并根据不同的交互模式（Mode）调用底层的 Agent 定义、工具和技能，完成具体的业务逻辑编排。该层实现了意图识别与具体执行的解耦。
1. quick_handler.py (快问快答模式)
- 职责：处理低延迟、直接的问答需求。
- 特性：
- - 通常关闭推理（Reasoning）。
- - 侧重于高效的 RAG（检索增强生成）和基础对话。
- - 适用于：常见问题查询、简单指令执行。
2. plan_handler.py (自规划模式)
- 职责：处理复杂、需要多步拆解的任务。
- 特性：
- - 默认开启推理（Reasoning/CoT）。
- - 允许 Agent 自主决定调用哪些工具（Tool Use）以及调用的顺序。
- - 适用于：深度数据分析、多步骤调研、逻辑推理题。
3. team_handler.py (团队协作模式)
- 职责：编排多个 Agent 协同工作。
- 特性：
- - 管理 Leader Agent 与 Member Agents 之间的任务分发（Delegation）。
- - 处理多智能体之间的上下文共享与任务交接（Handover）。
- - 适用于：复杂的软件开发流程、市场方案策划（包含文案、图片、数据多员协作）。
4. flow_handler.py (工作流模式)
- 职责：执行具有严格顺序和状态管理的长流程任务。
- 特性：
- - 基于 Agno 的 Workflow 类实现。
- - 支持状态持久化、步骤跳转和确定性的逻辑分支。
- - 适用于：审批流程、标准化的数据清洗管道、用户入职引导。
