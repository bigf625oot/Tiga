### 如何进行测试 (Testing Guide)
您现在可以通过以下步骤在本地环境中测试新功能：

1. 访问页面 :
   
   - 启动项目后，打开浏览器访问 http://localhost:5173/ (或您的实际端口)。
   - 在左侧导航栏（或移动端菜单）中，找到 "流水线" 分组，点击 "ETL流水线" （对应图标 Workflow ）。
   - 或者直接点击新建的 "ETL Editor" 入口（如果有直接链接）。根据 App.vue 的集成，它应该已经在动态组件切换列表中，key 为 etl_pipeline 。
2. 功能验证流程 :
   
   - 拖拽添加节点 :
     
     - 在编辑器左侧的 "组件库" 面板中，按住 Kafka Source 或 Redis Sink 。
     - 将其拖拽到中间的 画布区域 并释放。
     - 预期 : 画布上应出现一个新的卡片节点，带有相应的图标和标题。
   - 节点连线 :
     
     - 将鼠标悬停在 Kafka Source 节点的右侧连接点（Handle）。
     - 按住并拖动连接线到 Redis Sink 节点的左侧连接点。
     - 预期 : 一条带箭头的连线应成功创建，连接两个节点。
   - 运行流水线 :
     
     - 点击画布右上角的 "运行" (Play) 按钮。
     - 预期 :
       - 节点状态应变为 "Running"（绿色脉冲效果）。
       - 右上角按钮变为 "停止"。
       - 后端 API /pipelines/{id}/run 被调用（您可以在浏览器 Network 面板查看）。
   - 保存流水线 :
     
     - 点击右上角的 "保存" 按钮。
     - 预期 : 当前画布的节点布局和连线关系被保存到后端数据库。
### 代码变更摘要
- 入口集成 : 在 App.vue 中注册了 EditorLayout 组件，并添加了 etl_pipeline 视图映射。
- 侧边栏修复 : 修正了 Sidebar.vue 中的节点类型枚举，使其与后端定义 ( source , transform , sink ) 严格一致，解决了拖拽后节点类型错误的问题。
- 状态管理 : usePipelineStore 现已支持节点添加、删除及运行状态的实时响应。
- 运行逻辑修复 : 修复了在未保存流水线时点击运行无效的问题。现在点击运行会自动保存（新建）当前流水线并启动执行。
- 节点交互修复 : 
  - 修复了节点连接点 (Handle) 被遮挡导致无法连线的问题。
  - 优化了 VueFlow 实例初始化逻辑，解决了节点拖拽创建位置偏移和连线无响应的问题。
  - 规范了自定义节点的数据访问，增强了交互稳定性。
- 功能增强 :
  - 属性面板 : 实现了右侧属性配置面板。点击节点后，可配置 Kafka/Redis/Postgres 等组件的详细参数（如 Topic, Host, Port）。
  - 连线校验 : 增加了连线规则校验，禁止自环（Source=Target）和重复连线。
  - 状态监控 : 实现了流水线运行时的状态轮询，实时更新节点状态和指标。
